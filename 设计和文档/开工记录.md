# OpenGameTranslator 开工记录

这份文档用于在新窗口快速恢复项目上下文。优先记录项目目标、当前决策、已完成工作和下一步任务，不记录本地逆向报告的具体内容。

## 1. 项目目标

OpenGameTranslator 是一个开源游戏翻译 CLI 工具。

核心目标：

- 不内置翻译 API。
- 不做自动翻译。
- 普通用户不手动改游戏文件；优先非侵入式，复杂度明显更高时允许工具做可溯源、可恢复的托管修改。
- 支持文本提取、CSV 导出、译文校验、翻译包构建、运行时 Overlay。
- 给 LLM 使用的翻译文件使用 CSV，两列：`原文`、`译文`。
- 控制码、脚本代码、HTML tag 等应尽量从翻译文本中分离，避免 LLM 漏回写代码导致运行时回退原文。
- 如果外部翻译工具把译文覆盖到第一列，工具链需要能按原始 CSV 的行顺序恢复成 `原文`、`译文` 双列格式。
- 引擎适配器必须和 Core 层解耦。
- 代码要新手友好，方便有少量 C# 经验的维护者参与。

第一阶段目标是做最小可维护架构和项目骨架，不做大而全框架。

## 4. 计划中的最小架构

建议目录分层：

```text
src/
    core/
    engines/
        tyrano/
        cocos2d-js/
        rpgmaker-mv/
        rpgmaker-mz/
    cli/
runtime/
    tyrano/
    cocos2d-js/
    rpgmaker-mv/
    rpgmaker-mz/
```

Core 层负责引擎通用逻辑：

- `ExtractedTextEntry` 数据模型。
- `TranslationEntry` 数据模型。
- CSV 读写。
- 控制码和占位符校验。
- 翻译包构建。
- `Result<T>` 错误处理模型。
- 日志输出。

Engine Adapter 层负责具体引擎：

- 检测游戏是否属于该引擎。
- 提取该引擎文本。
- 提供运行时 Hook 文件模板。
- 提供该引擎需要保留的控制码规则。

CLI 层只负责命令入口：

- `detect`
- `extract`
- `cocos-export-csv`
- `validate`
- `repair`
- `build`
- `install`
- `install-cocos`
- `install-probe`
- `verify-install`
- `verify-cocos`
- `verify-probe`
- `uninstall`
- `uninstall-cocos`
- `uninstall-probe`

Runtime 层原则：

- 找到合法译文则显示译文。
- 找不到译文则显示原文。
- 译文校验失败则显示原文。
- 出现异常则显示原文。
- 用户侧不手动修改文件；当前 TyranoScript 先采用托管 patch，后续再评估非侵入式 launcher。

## 5. 当前关键决策

- 不内置翻译 API，这是明确边界。
- Runtime 只读取已经构建好的本地翻译包，不调用翻译服务。
- 翻译失败、未命中或运行时异常时，必须回退原文。
- 首选非侵入式；如果成本明显更高，先做托管式可恢复修改。
- `index.html` 手动修改只能作为开发期烟测；用户方案必须由工具自动安装和还原。
- 当前 TyranoScript 采用《背信少女》作为代表验证游戏；托管安装会备份原文件、写 manifest、生成 `run-me.bat` 和 `restore-original.bat`。
- 当前 Cocos2d-JS 采用运行时资源导出：启动游戏后通过 JSB 读取明文 `data/project.json`，再由 CLI 转 CSV。
- 当前 Cocos2d-JS 翻译 runtime 不修改 `Resources/data/project.json` 源文件，而是在 `getStringFromFile("data/project.json")` 返回前替换内存中的字符串值。
- Cocos2d-JS 探针工作目录暂放在 `Resources/OpenGameTranslator/`，因为 `require()` 更稳定地从 `Resources` 搜索路径加载运行时代码。
- Core 层不能依赖任何具体游戏引擎。
- 第一版优先实现 TyranoScript 适配器。
- 面向 LLM 的翻译输入输出使用双列 CSV：`原文`、`译文`。
- 提取阶段先把 Tyrano 控制 tag、HTML tag、特殊控制符从翻译文本中切开。
- 本地逆向报告只作为开发参考，不提交到 git 仓库。
- 类型检查按严格标准设计，让编译器尽量提前暴露问题。

## 8. 当前命令

安装依赖：

```bash
npm install
```

```bash
npm run typecheck
```

构建：

```bash
npm run build
```

开发运行：

```bash
npm run dev
```

检测本地游戏引擎：

```bash
npm run dev -- detect "<game-path>"
```

TyranoScript 代表游戏示例：

```bash
npm run dev -- detect "games/背信少女25.1.28补"
```

提取翻译 CSV：

```bash
npm run dev -- extract "games/背信少女25.1.28补" output/perfedygirl.csv
```

```bash
npm run dev -- validate output/perfedygirl.csv
```

```bash
npm run dev -- repair output/perfedygirl.csv output/perfedygirl_translated.csv output/perfedygirl.repaired.csv
```

构建 runtime 翻译包：

```bash
npm run dev -- build output/perfedygirl.repaired.csv output/perfedygirl.package.json
```

生成并校验托管安装：

```bash
npm run dev -- install "games/背信少女25.1.28补" output/perfedygirl.package.json perfedygirl01.exe
npm run dev -- verify-install "games/背信少女25.1.28补"
```

还原原文件：

```bash
npm run dev -- uninstall "games/背信少女25.1.28补"
```

Cocos2d-JS 探针检测：

```bash
npm run dev -- detect "games/maya"
```

Cocos2d-JS 探针安装、校验和还原：

```bash
npm run dev -- install-probe "games/maya" player.exe
npm run dev -- verify-probe "games/maya"
```

```bash
npm run dev -- cocos-export-csv "games/maya/Resources/OpenGameTranslator/output/opengametranslator-extracted-texts.json" output/maya.csv
```

Cocos2d-JS 翻译包构建和安装：

```bash
npm run dev -- validate output/maya.translated.csv
```
