# OpenGameTranslator 开工记录

这份文档用于在新窗口快速恢复项目上下文。优先记录项目目标、当前决策、已完成工作和下一步任务，不记录本地逆向报告的具体内容。

## 1. 项目目标

OpenGameTranslator 是一个开源游戏翻译 CLI 工具。

核心目标：

- 不内置翻译 API。
- 不做自动翻译。
- 普通用户不手动改游戏文件；优先非侵入式，复杂度明显更高时允许工具做可溯源、可恢复的托管修改。
- 支持文本提取、CSV 导出、译文校验、翻译包构建、运行时 Overlay。
- 给 LLM 使用的翻译文件使用 CSV，两列：`原文`、`译文`。
- 控制码、脚本代码、HTML tag 等应尽量从翻译文本中分离，避免 LLM 漏回写代码导致运行时回退原文。
- 如果外部翻译工具把译文覆盖到第一列，工具链需要能按原始 CSV 的行顺序恢复成 `原文`、`译文` 双列格式。
- 引擎适配器必须和 Core 层解耦。
- 代码要新手友好，方便有少量 C# 经验的维护者参与。

第一阶段目标是做最小可维护架构和项目骨架，不做大而全框架。

## 2. 技术和风格约束

技术栈：

- TypeScript + Node.js。
- CLI 工具。
- strict TypeScript，并尽量开启更严格的类型检查。
- 不做 GUI。
- 运行时插件可以是普通 JavaScript。

代码风格：

- 偏 C# / .NET 风格的 TypeScript。
- 优先使用 `class`、`interface`、`readonly`、`public`、`private`。
- 显式返回类型。
- 少用复杂箭头函数和匿名函数链式调用。
- 避免 `reduce`。
- 复杂逻辑拆成具名方法。
- 可以多用 `for...of`。
- 不做复杂 DI 容器。
- 不做抽象工厂套抽象工厂。
- 不滥用 `any`；未知 JSON 优先用 `unknown`，再做类型判断。
- 代码里面需要有适量注释，帮助新手理解关键思路；不要逐行解释，也不要堆太多注释。

## 3. 计划中的最小架构

建议目录分层：

```text
src/
    core/
    engines/
        tyrano/
        rpgmaker-mv/
        rpgmaker-mz/
    cli/
runtime/
    tyrano/
    rpgmaker-mv/
    rpgmaker-mz/
```

Core 层负责引擎通用逻辑：

- `ExtractedTextEntry` 数据模型。
- `TranslationEntry` 数据模型。
- CSV 读写。
- 控制码和占位符校验。
- 翻译包构建。
- `Result<T>` 错误处理模型。
- 日志输出。

Engine Adapter 层负责具体引擎：

- 检测游戏是否属于该引擎。
- 提取该引擎文本。
- 提供运行时 Hook 文件模板。
- 提供该引擎需要保留的控制码规则。

CLI 层只负责命令入口：

- `detect`
- `extract`
- `validate`
- `repair`
- `build`
- `install`
- `verify-install`
- `uninstall`

Runtime 层原则：

- 找到合法译文则显示译文。
- 找不到译文则显示原文。
- 译文校验失败则显示原文。
- 出现异常则显示原文。
- 用户侧不手动修改文件；当前 TyranoScript 先采用托管 patch，后续再评估非侵入式 launcher。

## 4. 当前关键决策

- 不内置翻译 API，这是明确边界。
- Runtime 只读取已经构建好的本地翻译包，不调用翻译服务。
- 翻译失败、未命中或运行时异常时，必须回退原文。
- 首选非侵入式；如果成本明显更高，先做托管式可恢复修改。
- `index.html` 手动修改只能作为开发期烟测；用户方案必须由工具自动安装和还原。
- 当前《背信少女》采用托管安装：备份原文件、写 manifest、生成 `run-me.bat` 和 `restore-original.bat`。
- Core 层不能依赖任何具体游戏引擎。
- 第一版优先实现 TyranoScript 适配器。
- 面向 LLM 的翻译输入输出使用双列 CSV：`原文`、`译文`。
- 提取阶段先把 Tyrano 控制 tag、HTML tag、特殊控制符从翻译文本中切开。
- 本地逆向报告只作为开发参考，不提交到 git 仓库。
- 类型检查按严格标准设计，让编译器尽量提前暴露问题。

## 7. 当前命令

安装依赖：

```bash
npm install
```

类型检查：

```bash
npm run typecheck
```

构建：

```bash
npm run build
```

开发运行：

```bash
npm run dev
```

检测本地游戏引擎：

```bash
npm run dev -- detect "games/背信少女25.1.28补"
```

提取翻译 CSV：

```bash
npm run dev -- extract "games/背信少女25.1.28补" output/perfedygirl.csv
```

校验翻译 CSV：

```bash
npm run dev -- validate output/perfedygirl.csv
```

恢复被翻译工具覆盖第一列的 CSV：

```bash
npm run dev -- repair output/perfedygirl.csv output/perfedygirl_translated.csv output/perfedygirl.repaired.csv
```

构建 runtime 翻译包：

```bash
npm run dev -- build output/perfedygirl.repaired.csv output/perfedygirl.package.json
```

生成并校验托管安装：

```bash
npm run dev -- install "games/背信少女25.1.28补" output/perfedygirl.package.json perfedygirl01.exe
npm run dev -- verify-install "games/背信少女25.1.28补"
```

还原原文件：

```bash
npm run dev -- uninstall "games/背信少女25.1.28补"
```

查看 git 状态：

```bash
git status --short --branch --ignored
```
